Hacking Guide

The monolog module is a logging application that attaches to the PSR-3 compatible Xoops Logger interface. The monolog module listens to events occuring in your XOOPS system, and feeds them to logs controlled by Jordi Boggiano's Monolog pacakge: https://github.com/Seldaek/monolog. This logger makes remote debugging available at a level never before possible with XOOPS.

The Monolog package provides a huge number of components which can be combined in numerous ways. The interface provided by this module is a simple one, and there is ample room for improvement. Ultimately, a more sophisticated configuration file based mechanism that supports the inherent flexibility of Monolog is planned.

Until that interface becomes available, it is possible to modify the module to fit your specific requirements. This guide is intended to point you in the right direction to implement the logging soulution you require.

class/monologlogger.php

The enable() method in class/monologlogger.php instantiates the basic Monolog. All Monolog classes are available by autoload, so any logging system it can create can be implemented here. There are some caveats to working at this level:

  • Resources may not be available - Logging begins before a database is available, before Locale is established, before -- well, you get the idea. Make sure your code does not break by depending on things that are not yet available.
